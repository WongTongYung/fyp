                                          ABSTRACT

         Pickleball is currently growing at an exponential rate globally. However, a major barrier has
emerged due to its unique officiating requirements of pickleball. Pickleball utilizes a complex side-out
scoring format where points are only awarded to the serving team, and the score is announced as a unique
three-number. Recreational players are subjected to a significant cognitive burden due to this non-linear
scoring structure, which frequently results in memory errors, on-court disputes, and a disturbed gameplay
flow. While professional tournaments benefit from high-fidelity optical tracking systems like Hawk-Eye,
these solutions rely on expensive multi-camera infrastructures that are financially and logistically
inaccessible to the average recreational community. Similar to this, current consumer solutions either have
ecosystem exclusivity issues or necessitate the purchase of expensive, specialized hardware, creating a need
for an affordable substitute. This project involves the creation of an affordable Automated Pickleball Point
Counting System that can run fully on a typical consumer laptop. The proposed solution integrates computer
vision and artificial intelligence to track match events in real-time using a standard USB webcam. The
methodology begins with the custom dataset comprising footage of local gameplay, which is annotated to
fine-tune the YOLOv11-m object detection model. The detected object coordinates are subsequently
processed by a custom Python-based Game Logic Engine. This engine operates as a finite state machine,
enforcing the server rotation and rally termination conditions. Data persistence is managed using local
SQLite database, which guarantees offline functionality and facilitates detailed post-match analytics. The
final system is accessible through a lightweight web-based dashboard that visualizes the live three-number
score and includes a video rewind feature for immediate dispute verification. By validating this prototype
against manual scoring methods, the project demonstrates that high-level sports analytics can be
democratized. Providing recreational players with a reliable, automated tool that enhances the enjoyment
of the sport.

1.1 Introduction
         Pickleball is a rapidly growing paddle sport that attracts a lot of newcomers due to its easy

learning curve. The origin of the pickleball was invented in 1965 in Washington, USA (Brinkerhoff,
2023). The game is a combination of few elements such as tennis, badminton, and ping pong. The
court is played on a badminton-sized court with a slightly lower net similar to tennis while using
an oversized paddle to hit a polymer ball. The game usually plays in doubles since it is the most
common format in both recreational and competitive play.

         Pickleball game consists of few serving rules to decide the scoring. The serves must starts
with underhand with the paddle below the waist. Ball must travel diagonally passing the non-volley
zone and bounce once by both sides. This rule is named as two-bounce rule. If a player volleys
within the kitchen, hits the ball into the net, or out of bounds. It is considered as fault that leads to
the end of the rally.

         The common scoring system in pickleball is known as traditional side-out scoring. This is
the primary scoring system adopted by the official tournament and used as a default in recreational
play. The scoring mechanics awards points to the serving side if wins a rally. But if the receiving
team wins, they will win a serve instead of the points (Mackie, 2024). A lot of confusion came from
the doubles scoring which requires tracking three numbers. Which is the serving teams score, the
receiving teams score, and the server number. Before serving, the server must announce these three-
number scores. This increases the difficulty for players to remember the numbers, especially when
there are long, intense rallies or distractions on or off the court.

1.2 Problem Statement
         Pickleball is the fastest-growing sport in Malaysia, primarily due to the low barrier of entry

and highly social sport. This has led to significant growth in new recreational and amateur players
in recent years (Elara, 2023).

         However, the supporting technology for the game has not kept pace with the current rapid
growth of players. For the vast majority of recreational and amateur matches, scorekeeping remains
a manual process. This system is highly prone to human error, indicating players are overload with
the information since players need to track the three-number score, server rotation, and game state
simultaneously while actively playing the pickleball.

         The unreliability of this manual process frequently leads to disputes over the score. These
conflicts not only disrupt the flow of play but also diminish the overall enjoyment of the players.
While professional-grade automated systems like Hawk-Eye exist in the market, these systems are
not available for vast majority of the pickleball community. They are purposely designed for
professional broadcast tournaments, not for recreational and community courts.

         Therefore, this project addresses the issue of lack of accessible technology between the
recreational and amateur pickleball community. The aim is to develop a low-cost, reliable, and

user-friendly automated point counting system for average players enhancing the players overall
enjoyment and experience.

1.3 Scope
         The scope of this project is to develop an Automated Pickleball Point Counting System for

recreational pickleball running on a local PC. The object detection module using YOLOv11-m
model and fine-tuning using the PyTorch framework. This model will be implemented in Python,
using the OpenCV library to process the video feed. This project will be using SQLite database to
store the game events with a timestamp. The simple graphical user interface to display in dashboard
will be built with standard HTML, CSS, and JavaScript, running in a browser.

         Due to the difficulty of tracking the player pose movement, this project will be limited to
tracking the ball only. The foot faults or kitchen violations will not be included in the system.
Currently, a single camera will be used to setup in the between on the court to detect the movement
of the ball.

1.5 Brief Methodology
         This project adopts the Waterfall model as the primary Software Development Life Cycle

(SDLC). This model follows the sequential approach for the development of stages, ensuring that
requirements data are developed and recorded before proceeding to implementation and testing.
The project execution is organized into the following four phases which are data collection and
preparation, computer vision model implementation, game engine and database construction and
system integration and testing. Figure 1.1 below show the Waterfall diagram in the sequential
approach.

                                                          4
                                     Figure 1.1 Waterfall model diagram

         The development process begins with data collection and preparation. A custom dataset will
be created by recording local pickleball games. Raw video footage will be processed into individual
frames, which will be annotated with bounding boxes to create training data.

         This next phase followed by the computer vision model implementation. YOLO object
detection model will be fine-tuned on the custom dataset using the PyTorch framework. The
training process in this stage will focus on maximizing mean Average Precision to ensure the model
reliably outputs accurate real-time ball coordinates.

         The third phase involves game engine and database construction. A Python-based game
logic engine will be designed to manage side-out scoring rules, track server rotation, and maintain
the overall game state. Additionally, an SQLite database will be implemented to log all game events
with timestamps.

         Finally, the process concludes with system integration and testing. The coordinates are fed
to the logic engine to update the score and log events, allowing for real-time dashboard display and
post-match analytics. This phase will be performed in real-time testing. System will be validated
in a live match environment to compare the scoring accuracy against manual scorekeeping.

1.6 Significance of Project
         The primary significance of this project is to enhance the player experience. By automating

the process of manual scoring, the system effectively reduces human error, which minimizes on-
court disputes and game interruptions. This allows players to remain fully focused on the game,
thereby improving the overall experience and enjoyment of the sport.

         Furthermore, the project plays a crucial role in democratizing sports technology. It
addresses a significant accessibility gap by demonstrating that a low-cost, software-based solution
provides a viable alternative to expensive professional systems. This approach makes advanced
sports analytics accessible to the wider recreational community.
